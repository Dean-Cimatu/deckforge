# DeckForge

Turn any lecture, slide deck, PDF, YouTube video, or article into flashcards you'll actually remember — with spaced-repetition built in.

## Features

- **Ingest anything** — plain text, PDF, images/slides, YouTube URLs, web articles
- **AI-powered** — Claude Haiku generates summaries and flashcards automatically
- **Spaced repetition** — SM-2 algorithm schedules reviews at the right time
- **PWA** — installable, works offline (review queue syncs when back online)
- **API keys** — programmatic access via `/api/v1`

## Local development

**Prerequisites:** Node 20, pnpm 9+, MongoDB (local or Atlas URI)

```bash
# 1. Clone
git clone https://github.com/Dean-Cimatu/deckforge.git
cd deckforge

# 2. Install deps
pnpm install

# 3. Copy env and fill in your keys
cp .env.example .env
# Set ANTHROPIC_API_KEY (required for generation)
# Set MONGODB_URI if not using local default
# JWT_SECRET defaults to a dev value — change it before deploying

# 4. Build shared types
pnpm --filter @deckforge/shared build

# 5. Start API + Web in separate terminals
pnpm dev:api   # http://localhost:3001
pnpm dev:web   # http://localhost:5173
```

### With Docker Compose

```bash
docker compose up
# API → http://localhost:3000
# Web → http://localhost:5173
```

## Deploying to Render

The repo ships a `render.yaml` that defines both services.

1. Push your repo to GitHub
2. In [Render](https://render.com), click **New → Blueprint** and connect this repo
3. Set the following environment variables in the Render dashboard:

| Service | Variable | Value |
|---|---|---|
| deckforge-api | `MONGODB_URI` | Your Atlas connection string |
| deckforge-api | `ANTHROPIC_API_KEY` | Your Anthropic key |
| deckforge-api | `CORS_ORIGIN` | Your web app URL (e.g. `https://deckforge-web.onrender.com`) |
| deckforge-web | `VITE_API_BASE` | Your API URL (e.g. `https://deckforge-api.onrender.com/api`) |

`JWT_SECRET` is auto-generated by Render (`generateValue: true`).

> **First deploy note:** Render free-tier services spin down after inactivity. The first request after a cold start takes ~30s.

## API reference (v1)

Authenticate with `Authorization: Bearer <your-api-key>` (create keys at `/settings`).

```
GET  /api/v1/sources
GET  /api/v1/decks
GET  /api/v1/review/queue         ?deckId=<id>  (optional)
POST /api/v1/cards/:id/review     { grade: 0|3|4|5 }
```

| Backend | Express 4, TypeScript, MongoDB/Mongoose |
| AI | Anthropic Claude Haiku |
| Auth | JWT, httpOnly cookies |
| Spaced repetition | SM-2 algorithm |
| PWA | vite-plugin-pwa, Workbox, idb-keyval |
| Deploy | Render (API via Docker, Web as static site) |
